# Troubleshooting Playbook

Quick reference for common issues and solutions.

## Format
Each issue gets its own file: `issue-description.md`

## Template
```markdown
# Issue: [Brief Description]

**Date First Encountered**: YYYY-MM-DD
**Severity**: Low/Medium/High
**Frequency**: One-time/Occasional/Frequent

## Symptoms
- What you observe
- Error messages
- Unusual behavior

## Diagnosis
1. Steps to identify root cause
2. Commands to run
3. What to check

## Solution
1. Step-by-step fix
2. Commands to run
3. Verification steps

## Prevention
- How to avoid in future
- Monitoring to set up

## References
- Related documentation
- External resources
```
